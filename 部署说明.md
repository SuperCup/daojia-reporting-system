# 到家平台提报管理系统 — 部署说明

## 系统概述

本系统为到家平台（美团/饿了么/淘宝等）的活动提报管理系统，包含运营端与客户端两个角色，支持活动提报全流程管理。

### 技术栈
- **后端**: Python 3.12 + FastAPI + SQLAlchemy + PostgreSQL
- **前端**: React 18 + Ant Design 5 + Vite
- **部署**: Docker Compose 容器化部署

### 功能模块

**客户端（提报人）**：
- 创建活动提报（活动时间、机制、区域、预算、券数量等）
- 勾选投放渠道（支持多选、按类型/名称筛选）
- 勾选投放商品（支持多选、按品牌/名称/UPC筛选）
- 渠道维护（增删改查）
- 商品维护（增删改查）
- 活动草稿/提交/查看状态

**运营端（管理员）**：
- 运营看板（活动统计、预算概览、待审核列表）
- 提报管理（查看全部提报、审核通过/驳回）
- 提报明细（渠道明细、商品明细、审核记录）
- 渠道管理（增删改查、批量操作）
- 商品管理（增删改查）
- 用户管理（创建账号、审核注册、禁用/启用）
- 费用管理（费用录入、费比追踪）
- 导出汇总（CSV格式）

**账号分发方式**：
- 方式一：管理员在用户管理页面批量创建账号并分发
- 方式二：客户端通过注册页面申请，管理员审核通过后激活

## 一、Docker Compose 一键部署（推荐）

### 前置条件
- Docker 20.10+ 及 Docker Compose 2.0+
- 服务器内存 ≥ 2GB
- 端口 80（前端）、8000（API）、5432（数据库）可用

### 步骤

1. **准备环境变量**

```bash
cp .env.example .env
```

编辑 `.env` 文件，修改以下关键项：
```ini
# 数据库密码（务必修改）
POSTGRES_PASSWORD=your_secure_password

# JWT密钥（务必修改为随机64字符）
SECRET_KEY=your_random_secret_key_here

# 初始管理员密码
INITIAL_ADMIN_PASSWORD=your_admin_password
```

2. **启动服务**

```bash
docker compose up -d --build
```

首次启动会自动：
- 创建 PostgreSQL 数据库
- 创建所有数据表
- 创建初始管理员账号（admin）
- 导入示例数据（平台、渠道、商品、示例活动）

3. **验证启动**

```bash
# 检查容器状态
docker compose ps

# 检查后端健康
curl http://localhost:8000/api/health

# 查看后端日志
docker compose logs backend
```

4. **访问系统**

| 地址 | 说明 |
|------|------|
| http://localhost | 前端界面 |
| http://localhost:8000/docs | API 文档（Swagger） |
| http://localhost:8000/api/health | 健康检查 |

5. **登录系统**

```
管理员: admin / admin123456（或你在.env中设置的密码）
客户端: client_jilin / client123456
```

## 三、生产环境部署

### 1. 服务器准备

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo systemctl enable docker
sudo systemctl start docker

# 安装 Docker Compose
sudo apt-get install -y docker-compose-plugin
```

### 2. 部署应用

```bash
# 上传项目文件
scp -r daojia-reporting-system/ user@server:/opt/

# 在服务器上
cd /opt/daojia-reporting-system
cp .env.example .env
# 编辑 .env，设置强密码和密钥

# 构建并启动
docker compose up -d --build
```

### 3. 配置 Nginx 反向代理（可选，用于域名和HTTPS）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端
    location / {
        proxy_pass http://127.0.0.1:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # API
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

使用 Let's Encrypt 配置 HTTPS：
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 4. 数据库备份

```bash
# 手动备份
docker exec daojia-db pg_dump -U postgres daojia_reporting > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i daojia-db psql -U postgres daojia_reporting < backup_20260101.sql
```

### 5. 设置自动备份（crontab）

```bash
# 编辑定时任务
crontab -e

# 每天凌晨2点备份
0 2 * * * docker exec daojia-db pg_dump -U postgres daojia_reporting > /backups/daojia_$(date +\%Y\%m\%d).sql
```

## 五、数据库说明

### 核心表结构

| 表名 | 说明 |
|------|------|
| users | 用户表（管理员/运营/客户端） |
| platforms | 平台表（美团/饿了么/淘宝等） |
| channels | 渠道表（店铺/连锁，关联平台） |
| products | 商品表（UPC/名称/品牌系列） |
| activities | 活动提报表（核心业务表） |
| activity_channels | 活动-渠道关联表（多对多） |
| activity_products | 活动-商品关联表（多对多） |
| cost_records | 费用记录表 |

### 活动状态流转

```
草稿(draft) → 已提交(submitted) → 已通过(approved)
                               → 已驳回(rejected) → 可重新编辑提交
```

## 八、常见问题

**Q: 忘记管理员密码？**
A: 进入数据库直接修改：`docker exec -it daojia-db psql -U postgres daojia_reporting -c "UPDATE users SET password_hash = '\$2b\$12\$...' WHERE username='admin'"` 或重置环境变量后重建。

**Q: 如何修改CORS配置？**
A: 编辑 `.env` 文件中的 `CORS_ORIGINS`，重启后端。

**Q: 数据库如何迁移？**
A: 使用 `pg_dump` 导出，在新环境 `psql` 导入。

**Q: 如何添加新的渠道类型？**
A: 在渠道管理页面创建渠道时直接输入新的渠道类型即可。

**Q: 前端构建失败？**
A: 确保Node.js版本 ≥ 18，使用 `npm install --legacy-peer-deps` 解决依赖冲突。
